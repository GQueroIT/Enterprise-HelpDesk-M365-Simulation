# Jira Configuration

## Purpose

This file documents the initial configuration of Jira Service Management for the Enterprise Help Desk & Microsoft 365 Simulation project.

The goal of this setup was to establish a functional Help Desk ticketing environment that could be used to simulate real-world IT support workflows.

## Platform

- Product: Jira Service Management
- Role in Project: Active ticketing platform
- Use Case: Help Desk workflow simulation, issue tracking, investigation, and resolution

## Initial Configuration

The Jira environment was configured to support a basic IT support workflow.

### Core setup completed

- Jira Service Management instance created
- IT support-focused project selected during setup
- Basic service management template used
- Initial ticketing workflow enabled
- Ticket lifecycle documentation structure created in the repository

## Operational Scope

This Jira setup is being used to support the following project goals:

- simulate Help Desk ticket intake
- document issue investigation
- track troubleshooting steps
- record ticket resolution
- organize evidence by support phase

## Initial Workflow Structure

The first Jira workflow implemented in this project follows this pattern:

1. Ticket creation
2. Issue simulation
3. Investigation
4. Resolution

This structure is intentionally simple at this stage so the project can focus on building realistic support scenarios before expanding into more advanced workflows.

## Project Integration

Jira is currently integrated into the project as the active ticketing system for:

- initial Help Desk ticket lifecycle simulation
- structured evidence collection
- workflow documentation
- issue tracking tied to the Active Directory lab environment

## Future Expansion

This Jira environment is planned to expand into:

- additional incident scenarios
- access issues
- account lockouts
- onboarding workflows
- troubleshooting playbooks
- escalation simulation
- automation-linked workflows

## Notes

Jira is currently the active ticketing system in the project.

ServiceNow is planned as the second system and will later be added as the primary enterprise-style ITSM platform for comparison and workflow expansion.
