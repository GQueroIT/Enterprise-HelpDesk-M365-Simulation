# Jira Setup Notes

## Overview

This file contains notes related to the initial Jira Service Management setup for the Enterprise Help Desk & Microsoft 365 Simulation project.

The purpose of this phase was to move from infrastructure-only work into actual Help Desk workflow simulation.

## Why Jira Was Added

After completing the core Active Directory environment, a ticketing platform was needed to simulate how IT support requests are received, investigated, documented, and resolved in a structured environment.

Jira was selected as the first active ticketing system because it provided a fast way to begin building realistic workflows and documenting ticket activity.

## What Was Completed

The Jira setup phase included:

- creating the Jira Service Management environment
- selecting the appropriate IT support service configuration
- creating the initial service project
- generating the first Help Desk ticket
- creating a basic incident workflow structure
- organizing evidence and notes by ticket lifecycle stage

## Initial Ticket Workflow

The first ticket followed a simple but realistic support path:

- step 01 - Jira setup
- step 02 - issue simulation
- step 03 - investigation
- step 04 - resolution

This structure was used to begin modeling how a Help Desk analyst would handle a support issue from beginning to end.

## Documentation Strategy

The setup was documented using separate folders for:

- configuration
- notes
- evidence
- troubleshooting
- workflow documentation

This was done to keep the project organized and scalable as more ticket scenarios are added.

## Project Value

Adding Jira changed the project from being only an infrastructure build into a more realistic IT support simulation.

It introduced:

- ticket lifecycle awareness
- issue tracking
- structured documentation
- support workflow thinking
- operational evidence collection

## Next Direction

The Jira setup phase now serves as the foundation for:

- more advanced Help Desk tickets
- Active Directory-related issue simulation
- ticket-based troubleshooting
- later integration with ServiceNow as a second system
